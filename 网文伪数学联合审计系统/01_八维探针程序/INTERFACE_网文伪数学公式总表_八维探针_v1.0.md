# 《网文伪数学公式总表》—《八维伪线性探针》接口映射 v1.0

## 1. 定位

两套系统不得合并为一个总分：

```text
公式总表：广域诊断、硬错误、AI味、文学表现、平滑化、连续性等
八维探针：商业执行专项评分、线性贡献、商业六门
接口层：证据路由、权限隔离、联合裁决
```

## 2. 固定执行顺序

```text
原文
→ 公式总表硬闸门
→ 广域风险向量
→ 八维探针提取与评分
→ 联合裁决
```

硬闸门失败时停止八维等级外推。可以继续提取八维作为诊断材料，但不得输出“稳健通过”。

## 3. 映射表

| 公式总表指标 | 八维探针目标 | 接口方式 | 限制 |
|---|---|---|---|
| 因果闸门、逻辑跳跃率 | 因果代价轴 | 先阻断，后作辅助证据 | 因果错误不得被八维高分补偿 |
| 人物主动度、动机断裂率 | 活人冲突轴、主角发动机门 | 证据输入 | 人物一致性失败先阻断 |
| 章末牵引度 | 章尾钩子轴 | 直接证据 | 不能把悬而未决自动当强钩子 |
| 当章回报度 | 最小兑现门 | 商业门证据 | 回报不等于主角必须获胜 |
| 情节平滑化 | 期待违背轴、极速流转轴 | 辅助诊断 | 不直接加减activation，避免重复计权 |
| 设定说明负载、规则调用 | 设定动作化轴 | 证据输入 | 规则矛盾仍由连续性闸门处理 |
| 配角反应、制度成本 | 侧面烘托轴 | 证据输入 | 旁观者惊叹不等于有效烘托 |
| 反差与人物稳定行为 | 反差造梗轴 | 证据输入 | 纯语气反差不计高分 |
| 主线可读性、新变量负载 | mainline.readability_risk | 等级上限 | 高负载需结合关联率判断 |
| AI味、AI参与伪概率 | 独立旁路 | 只报告 | 不进入八维总分 |
| 文学表现剖面 | 独立旁路 | 只报告 | 文学性不等于商业执行 |
| 肉身经验缺失 | 独立旁路 | 只报告 | 可作为表达修复，不进入activation |
| 留白不足 | 独立旁路/期待违背辅助 | 只作证据 | 不得一项风险双重扣分 |

## 4. 联合裁决矩阵

| 广域硬闸门 | 广域风险 | 八维等级 | 联合结论 |
|---|---|---|---|
| 失败 | 任意 | 任意 | 阻断；先修硬错误 |
| 通过 | 低/中 | 高 | 商业执行强，可进入定向表达修复 |
| 通过 | 高 | 高 | 执行强但病灶显著；不得直接发布 |
| 通过 | 低/中 | 低 | 基础稳定，但缺少追读发动机 |
| 通过 | 高 | 低 | 结构和表达同时需要重构 |
| 证据不足 | 任意 | 任意 | 补证据，不输出总判断 |

## 5. 去重与权限规则

1. 不建立“广域风险＋八维activation”的总分。
2. 同一原句最多作为两个高分轴的共同证据。
3. 同一病灶只设一个主轴，其他轴记录为连带轴。
4. AI味、文学性、肉身经验和留白是旁路，不进入八维线性头。
5. 因果、人物、连续性、视角硬闸门具有最高优先级。
6. 平滑化高风险只能提示检查期待违背与流转，不能直接扣两次分。
7. 所有输出必须保留原文锚点、版本、哈希、输入尺度与校准状态。
