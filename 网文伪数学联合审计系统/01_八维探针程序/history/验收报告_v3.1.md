# 八维伪线性探针 v3.1 · 修复验收报告

## 验收对象

从 v3.0 的“Python 再次调用 API”架构，修正为：

```text
Codex 负责特征提取
Python 负责无 API 本地计算
```

## 结论

```text
修复项验收：通过
稳定版验收：不通过
当前状态：v3.1 候选版
```

## 通过项

| 验收项 | 结果 | 证据 |
|---|---|---|
| Python 不调用 API | 通过 | 无 OpenAI SDK、Key、模型、网络调用 |
| 无第三方依赖 | 通过 | 仅 Python 3.11+ 标准库 |
| Codex 与脚本职责分离 | 通过 | `prepare` 生成任务；Codex 生成 `extraction.json`；`score` 本地计算 |
| 正文与提取结果绑定 | 通过 | task_id + SHA-256 + input_type + 版本锁 |
| 八维完整性 | 通过 | 缺失、重复、非法值直接阻断 |
| 证据不足处理 | 通过 | `score=null`，不按 0 分处理 |
| 合格分映射 | 通过 | 八维全 2 分 → activation 0.5 → 中 |
| 商业门控 | 通过 | 关键门低于 2 时禁止高/中 |
| 报告生成 | 通过 | JSON 与 Markdown 同时生成 |
| 批量计算 | 通过 | 示例 2/2 成功 |
| 修改前后对比 | 通过 | 兼容性检查与轴差值生成 |
| 权重拟合代码路径 | 通过 | 40 条合成样本完整运行 |

## 未通过项

| 验收项 | 结果 | 原因 |
|---|---|---|
| 用户真实 Codex 项目端到端 | 待验 | 当前环境未运行用户的 Codex 项目 |
| 十题材稳定性 | 未完成 | 尚未提供 10 段真实测试集 |
| 重复提取一致性 | 未完成 | 尚未进行多轮 Codex重复执行 |
| 市场预测有效性 | 不成立 | 本工具没有真实平台行为数据 |
| 默认权重有效性 | 未证明 | 默认权重仍为人工工程初始值 |

## 最终边界

v3.1 可以进入用户 Codex 项目进行候选测试，但不能宣称：

```text
真实线性探针
爆款预测模型
留存预测模型
已校准商业评分器
稳定版
```
