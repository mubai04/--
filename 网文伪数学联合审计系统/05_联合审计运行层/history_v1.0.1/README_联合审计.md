# 联合审计运行说明 v1.0.1 CANDIDATE

本版本保留原有全部模块，并修复输入真实性、严格类型校验、章级裁决优先级、商业六门自检和证据缺失问题。

完整输出顺序：

```text
输入完整性 → 直接测量 → 四硬闸门 → 章级六维 → 文学表现 → AI味 → 情节平滑化 → 九维风险 → AI参与伪概率 → 八维探针 → 商业六门 → 联合裁决
```

## 运行

```powershell
python 联合审计器.py self-test
python 联合审计器.py measure --source 测试样本/第一章_第三扇门不能开.md
python 联合审计器.py score --source 测试样本/第一章_第三扇门不能开.md --broad 测试样本/第一章_广域审计.json --probe 测试样本/第一章_八维探针摘要.json --output 测试报告/第一章_完整联合审计报告.json
```

## 输入绑定

正文、广域审计和八维摘要必须具有相同文件名与 SHA-256。正文发生任何改动后，旧JSON自动失效，必须重新提取。

## 严格规则

- 硬闸门“通过”只接受 JSON `true`/`false`。
- 所有广域评分必须包含分数、正文证据和判定。
- 八维必须恰好8轴，商业门必须恰好6门。
- activation必须在0—1；分数必须在0—4；置信度必须在0—100。
- 基础文本测量由本地脚本直接读取正文计算，人工记录不一致时拒绝运行。
- 章级状态未稳健通过时，八维高分不得把联合结论改写为“结构通过”。

## 边界

广域特征仍由Codex或人工提取，本地脚本负责直接测量、严格校验、公式计算和联合裁决，不调用API。AI参与百分比是C0伪概率，不是真实作者概率。
