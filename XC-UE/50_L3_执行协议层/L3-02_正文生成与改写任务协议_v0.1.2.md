# L3-02 正文修复任务规划协议 v0.1.2

> 状态：候选真源 / 未定版 / 待测试  
> 定位：规定候选正文生成、扩写、改写、局部修复需求如何被规划成任务包；L3 不执行正文生成或改写。  

---

## 1. 基本边界

L3-02 可以规划正文相关任务，但不判断正文好坏，也不修改正文。

正文好坏由 L1 / L2 判断。

L3-02 只回答：

- 哪个章节或段落需要后续执行器处理
- 根据哪个修复单改
- 读取哪些 IR 输入
- 目标文件引用是什么
- 任务约束是什么
- 完成后回哪里复验

---

## 2. 权限降级原则

v0.1 中的“新章生成”统一降级为：

```text
候选正文生成任务规划
```

v0.1 中的“章节扩写 / 局部改写”统一降级为：

```text
候选正文扩写任务规划 / 候选正文改写任务规划 / 指定范围局部修复任务规划
```

默认不得直接覆盖：

```text
chapters/ch01.md
chapters/ch02.md
```

默认只引用目标位置，不落候选正文。

---

## 3. 正文任务类型

允许任务：

1. 候选正文生成任务规划
2. 候选章节扩写任务规划
3. 候选局部改写任务规划
4. 删除冗余候选版任务规划
5. 补章末钩子任务规划
6. 补状态继承任务规划
7. 根据 L2 修复单规划指定段落修复
8. 多候选版本任务规划

禁止任务：

1. 无任务单自由写文。
2. 无 IR 输入自由写文。
3. 覆盖、备份或改写原文。
4. 改动未指定章节。
5. 用当前灵感覆盖 L2 修复方向。
6. 让正文改写反向修改系统规则。
7. L3 任务包写入后直接标记正文通过。
8. 声明候选正文已生成、diff 已生成、回填已完成。

```yaml

  任务编号: L3-TEXT-0001
  任务类型: 候选正文生成任务规划 / 候选章节扩写任务规划 / 候选局部改写任务规划 / 删除冗余规划 / 补钩子规划 / 补继承规划 / 多候选版本规划
  来源修复单: ""
  IR输入文件: []
  目标章节: chapters/ch01.md
  修改范围: 全章 / 指定段落 / 指定行 / 指定场景
  必须保留: []
  必须删除: []
  必须新增: []
  禁止项: []
  字数范围: ""
  输出方式: 仅生成任务包
  任务包文件: 运行记录/<run>/第三层/分项任务/<task>.md
  是否允许覆盖正式正文: 否
  回流验收位置: L1-01 / L1-02 / L1-03 / L1.5
```

---

## 5. 任务包规则

默认输出：

```text
运行记录/<run>/第三层/任务包.json
运行记录/<run>/第三层/分项任务/<task>.md
```

任务包必须写：

```text
来源修复单：
读取 IR：
目标文件引用：
任务约束：
复验入口：
```

---

## 6. 回流规则

后续执行器完成正文处理后必须回流；L3 本身只记录复验入口：

- 结构类修复 → L1-01
- 文风类修复 → L1-01
- 角色类修复 → L1-01
- 设定类修复 → L1-01
- 市场体验类修复 → L1-02 / L1-03
- 系统一致性修复 → L1-00 / L1-01 / L1.5

L3 任务包写入成功只代表任务规划完成，不代表正文已修复，也不代表 L1 / L2 判断通过。
