# XC-UE Codex 协作规则

- 所有新增项目文件名优先使用中文，配置文件和工具约定文件除外。
- Front Matter 是规则文档的人工元数据源；不得再依赖已删除的派生真源清单。
- 当前盘查基线是 `XC-UE_项目全量问题盘查报告_2026-06-23.md`；当前整改入口是 `XC-UE_P0整改任务单_2026-06-23.md`。
- 运行记录、归档、缓存、旧报告和工具输出禁止进入默认索引。
- 标准测试命令：`python -m pytest -q`。
- 当前无独立 Markdown 真源治理命令；以 `python -m pytest -q` 验证工程链路。
- 任何功能修复必须同步代码、Markdown 文档、自动测试和当前证据。
- 生产模式不得读取无真源资格规则；候选规则只能在显式候选试验模式下参与验证。
- 不得修改小说正文内容，不得新增 L4，不得用旧报告或旧测试输出冒充当前证据。
- 上述“不得修改小说正文内容”指 Codex 在开发过程中不得手工编辑、改写或替换小说正文；不禁止 B2 产品运行时在人工审批明确通过、补丁内容已冻结、`source_sha256`/`candidate_sha256`/`patch_sha256` 全部匹配、目标仍为 `TP-001`、操作仍为 `L2-01` 的 `REPLACE`/`APPEND`、且候选正文已经通过 L1 复验后，对正式正文执行原子应用或自动回滚。
- 未经批准不得修改正式正文仍为硬闸门；任何运行时正式应用都不得调用模型、API 或生成式改写。
