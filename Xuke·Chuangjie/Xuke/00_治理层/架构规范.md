# 架构规范

## 层级职责

每一层只做一件事，禁止跨层存取：

| 层级 | 允许 | 禁止 |
|------|------|------|
| 00_治理层 | 规范、规则、版本策略 | 业务数据 |
| 01_知识模型层 | Schema 定义 | 实例数据 |
| 02_经验资产库 | 经验原子实例 | 剧情正文 |
| 03_模式系统 | 组合模板与策略 | 单条孤立经验 |
| 04_读者模型 | 模型与公式 | 具体章节内容 |
| 05_评估系统 | 评估规则与输出格式 | 修改经验库 |
| 06_失败系统 | 失败记录与修复策略 | 成功案例 |
| 07_经验提取系统 | 提取流程与规则 | 最终入库（需审核） |
| 08_训练进化系统 | 优化、合并、版本 | 直接覆盖无版本 |
| 09_接口层 | 协议与调用说明 | 实现逻辑 |
| 10_决策系统 | 冲突矩阵、权重、推荐引擎、快照 | 正文生成 |

## 扩展规则

1. 新增经验类别须先更新 `01_知识模型层` Schema，再建目录
2. 所有经验原子必须有唯一 `经验ID`，格式：`XK-{类别缩写}-{序号}`
3. 破坏性变更必须升主版本，见 `版本管理规则.md`

## 语言规范

- 目录名、文件名、字段名、文档正文：**全部中文**
- 技术标识（经验ID、版本号）可含英文缩写
- 禁止中英混用字段（如 `reader_state` 与 `读者状态` 并存）
