# 版本管理规则

## 系统版本

- 当前系统：**v1.0**
- 格式：`主.次.修订`
  - **主**：架构或 Schema 不兼容变更
  - **次**：新增能力或大量资产
  - **修订**：修正、补充、小范围优化

## 经验原子版本

每条经验原子含 `版本` 字段：

```yaml
版本: "1.0"
```

变更类型：

| 变更 | 版本 bump | 说明 |
|------|-----------|------|
| 修正笔误 | 修订日志 | 不升版本 |
| 补充风险/标签 | 次 +1 | 1.0 → 1.1 |
| 修改触发条件或结果 | 主 +1 | 1.x → 2.0 |
| 废弃 | 标记 `状态: 已废弃` | 不删除，保留引用链 |

## 变更记录

重大变更在 `08_训练进化系统/版本进化/` 留档，含：

- 变更日期
- 变更原因
- 影响范围
- 旧版备份路径

## Git 约定（建议）

- 治理层 / Schema 变更：单独 commit
- 批量新增经验：按类别分批 commit
- commit message 使用中文，格式：`[Xuke] 动词 + 简述`
