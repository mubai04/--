# Xuke 架构说明

## 项目定义

**Xuke = 叙事经验操作系统**

### 职责

- 提炼小说经验
- 结构化叙事规律
- 管理读者心理模型
- 评估故事效果
- 积累可复用叙事资产

### 不负责

- 写正文小说
- 管剧情
- 管角色
- 管世界观执行

以上属于 **Chuangjie（工程执行层）**。

---

## 总体架构

```
Xuke/
│
├─ 00_治理层          规则中心：怎么建、怎么改、怎么扩展
├─ 01_知识模型层      数据结构定义（只定义形态，不存数据）
├─ 02_经验资产库      核心叙事经验资产
├─ 03_模式系统        经验组合与叙事策略
├─ 04_读者模型        阅读体验曲线操控
├─ 05_评估系统        章节与节奏效果评估
├─ 06_失败系统        失败模式记录与修复
├─ 07_经验提取系统    小说 → 经验原子
├─ 08_训练进化系统    经验库持续优化
├─ 09_接口层          供 Chuangjie 调用
├─ 10_决策系统        冲突 / 权重 / 推荐 / 快照（v1.1）
└─ 文档中心           愿景、架构、路线图、术语
```

---

## 核心原则

Xuke 不做三件事：

- ❌ 不写小说
- ❌ 不生成剧情
- ❌ 不管理世界

它只做一件事：

- ✔ 把「写小说的经验」变成结构化系统

---

## 与 Chuangjie 的边界

| 层级 | 系统 | 职责 |
|------|------|------|
| 经验层 | Xuke | 叙事规律、读者模型、评估、资产 |
| 执行层 | Chuangjie | 正文、剧情、角色、世界观执行 |

接口定义见 `09_接口层/`。
