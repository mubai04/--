# 经验提取流程（拆书流程）

> 作用：把小说 → 经验原子

---

## 流程总览

```
来源选定 → 范围划定 → 结构化拆解 → 经验提炼 → 去剧情化 → 审核 → 入库
```

---

## 阶段一：来源选定

| 来源类型 | 优先级 | 说明 |
|----------|--------|------|
| 爆款小说 | 高 | 经市场验证 |
| 编辑反馈 | 高 | 直接对应读者流失点 |
| 读者评论 | 中 | 需去噪 |
| 自己作品 | 中 | 可复盘 |

记录至 `来源库/{来源ID}.yaml`：书名、章节范围、提取日期、提取人。

---

## 阶段二：范围划定

- 单次提取范围：**1–3 章** 或 **1 个完整叙事弧**
- 必须标注：起止位置、核心事件（一句话，不含剧透式细节）

---

## 阶段三：结构化拆解

使用 `拆书模板/标准拆书模板.md` 填写：

1. **叙事结构**：压制/铺垫/爆发/收束 各占比例
2. **读者曲线**：读完后 Q/H/T/S/F 的定性变化（升/降/平）
3. **有效手法**：列出 1–3 个可复用手法
4. **失效点**：若有，记录失败类型

---

## 阶段四：经验提炼

将手法转化为经验原子：

```
手法描述 → 抽象为「条件 → 结果 → 读者效果」
         → 对照 01_知识模型层/经验原子.schema.yaml 填表
         → 分配经验ID
```

**提炼原则：**

- 去掉人名、地名、具体道具
- 保留因果结构
- 读者心理效果必须可量化（± 整数）

---

## 阶段五：去剧情化

检查清单：

- [ ] 文中无具体角色名
- [ ] 无世界观专有名词
- [ ] 脱离原书仍成立
- [ ] 触发条件可迁移到其他品类

不通过则退回阶段四。

---

## 阶段六：审核

按 `审核规则/经验审核清单.md` 执行：

- 完整性
- 无冲突（与已发布经验比对）
- 来源标注
- 状态改为「审核中」→ 审核人签字 → 「已发布」

---

## 阶段七：入库

- 文件放入 `02_经验资产库/{类别}/`
- 若形成多原子组合，额外在 `03_模式系统/模式链/` 建链
- 更新 `08_训练进化系统/版本进化/` 日志

---

## 产出时效参考

| 范围 | 预计耗时 |
|------|----------|
| 1 章 | 30–60 分钟 |
| 1 弧 | 2–4 小时 |
| 1 本精选 | 分批，按卷 |

---

## 与 Cursor 协作建议

1. 在 Cursor 中打开 `拆书模板/标准拆书模板.md` 副本
2. 粘贴原文片段（仅本地，不入库）
3. 让 Agent 按本流程输出 YAML 草稿
4. 人工审核后移入 `02_经验资产库`
