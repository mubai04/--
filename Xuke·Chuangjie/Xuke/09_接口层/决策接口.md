# 决策接口

> Xuke v2 完整决策流水线入口  
> 输出：**章节策略**（非正文），供 Chuangjie 执行

---

## 用途

根据章节上下文 + 读者状态 + 历史快照，自动完成：

1. 候选筛选
2. 冲突评估
3. 权重计算
4. Top3 经验 + 主模式链推荐
5. 叙事四相计划 + 风险警告

---

## 请求

```yaml
操作: 决策
协议版本: "2.0"

输入:
  章节上下文:
    章节ID: ""
    章节阶段: 起势 | 压制 | 转折 | 高潮 | 收束
    当前事件: []
    核心冲突: []
    剧情意图: ""

  读者状态:
    期待值: 0-100
    好奇心: 0-100
    紧张感: 0-100
    爽感储备: 0-100
    疲劳值: 0-100

  历史快照:
    最近模式链: []
    已用经验: []
    失败模式: []
    最近叙事弧: []

  世界状态:          # 可选
    资源: []
    阵营: []
    力量平衡: []
```

---

## 响应

```yaml
协议版本: "2.0"

决策输出:
  选定策略:
    主模式链: ""
    支撑经验: []      # Top3，已冲突校验

  叙事计划:
    压制相: []
    触发相: []
    爆发相: []
    收束相: []

  读者曲线预测:
    期待值: 0         # 变化量
    好奇心: 0
    紧张感: 0
    爽感储备: 0
    疲劳值: 0

  风险警告:
    等级: 低 | 中 | 高
    说明: []

  降级策略:
    疲劳过高时: ""
    悬念不足时: ""
    爽感不足时: ""

推荐明细:            # 可选展开
  - 经验ID: ""
    最终得分: 0
    推理说明: ""
```

## 执行规范

| 步骤 | 文件 |
|------|------|
| 输入校验 | `10_决策系统/模板/01_输入层.schema.yaml` |
| 候选筛选 | `10_决策系统/推荐引擎/决策流程.md` Step 2 |
| 冲突评估 | `10_决策系统/经验冲突矩阵.yaml` |
| 权重计算 | `10_决策系统/经验权重函数.md` |
| 输出组装 | `10_决策系统/模板/05_决策输出层.schema.yaml` |

完整示例：`10_决策系统/模板/示例/决策运行示例.yaml`

---

## 与推荐接口的区别

| 接口 | 粒度 |
|------|------|
| `推荐接口.md` | 轻量 Top 推荐列表 |
| `决策接口.md` | 完整六层流水线 + 叙事计划 + 预测 |

Chuangjie **规划阶段**用决策接口，**快速查单条经验**用推荐接口。
