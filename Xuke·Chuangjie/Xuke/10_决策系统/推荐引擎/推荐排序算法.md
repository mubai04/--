# 推荐排序算法

> Xuke v2 决策核心 · 正式工程化算法  
> 归属：**仅 Xuke `10_决策系统`**

---

## 算法目标

输入：章节上下文 + 读者状态 (Q,H,T,S,F) + 历史快照  
输出：Top3 经验原子 + Top1 模式链 + 叙事四相计划 + 读者曲线预测 + 风险警告

---

## 伪代码（可编排执行）

```
function 决策(输入):
    // ── Phase 0: 标准化 ──
    输入 ← 校验Schema(输入)
    输入.经验ID ← 别名映射为XK标准ID(输入)

    // ── Phase 1: 熔断（最高优先级）──
    熔断结果 ← 检查熔断(输入.读者状态, 输入.历史快照)
    if 熔断结果.触发:
        return 组装熔断输出(熔断结果)   // 跳过权重，直接输出

    // ── Phase 2: 候选生成 ──
    候选原子 ← []
    for each 经验 in 经验资产库 where 状态==已发布:
        Match ← 计算基础匹配分(经验, 输入)
        if Match == 0: continue
        候选原子.append({经验, Match})

    候选链 ← 筛选可用模式链(输入.历史快照, 输入.章节阶段)

    // ── Phase 3: 冲突评估 ──
    for each a in 候选原子:
        a.Conflict ← 查读者状态冲突(经验冲突矩阵, a, 输入.读者状态)
        a.Conflict += max(原子对冲突(候选原子, a))  // 预组合冲突

    // ── Phase 4: 权重计算 ──
    for each a in 候选原子:
        a.ReaderFit ← 读者适配系数(a, 输入.读者状态)
        a.Stage     ← 阶段系数(a, 输入.章节阶段)
        a.History   ← 历史惩罚(a, 输入.历史快照)
        a.Score ← clamp(
            a.Match × a.ReaderFit × a.Stage × a.History − a.Conflict,
            0, 100
        )

    for each chain in 候选链:
        chain.Score ← 模式链得分(chain, 候选原子)

    候选原子.sort(降序, key=Score)
    候选链.sort(降序, key=Score)

    // ── Phase 5: Top3 贪心选取 ──
    Top3 ← []
    for each a in 候选原子:
        if len(Top3) >= 3: break
        if a.Conflict >= 70: continue
        if 与Top3任一同章禁止(a, Top3): continue
        Top3.append(a)

    主链 ← 候选链[0] if 候选链 else null

    // ── Phase 6: 叙事计划组装 ──
    四相 ← 映射四相(Top3, 主链)
    预测 ← 叠加读者心理效果(Top3, 输入.读者状态)
    风险 ← 生成风险警告(Top3, 输入.读者状态, 冲突分析)
    降级 ← 生成降级策略(输入.读者状态)

    return 决策输出(Top3, 主链, 四相, 预测, 风险, 降级)
```

## Phase 1 · 熔断检查

```
function 检查熔断(读者状态, 历史):
    if 读者状态.F > 70:
        return {触发: true, 推荐: [XK-SP-001, XK-SJ-003], 禁止: [XK-YL-001]}
    if 读者状态.S > 80:
        return {触发: true, 推荐: [XK-GC-001, XK-SP-001], 禁止: [XK-YL-001, XK-XN-001]}
    if 读者状态.H < 20 and 历史.连续低好奇 >= 2:
        return {触发: true, 推荐: [XK-XN-001, XK-FZ-001]}
    if 历史.同卷GC001次数 >= 2:
        return {触发: true, 推荐: [XK-SP-001], 禁止: [XK-GC-001]}
    return {触发: false}
```

## Phase 2 · 基础匹配分

```
Match = 50
      + (满足前置数 / 前置总数) × 25
      + (满足触发数 / 触发总数) × 25

硬性触发不满足 → Match = 0
```

硬性触发示例（从经验 YAML 解析）：

- `S > 60` 且当前 S ≤ 60 → 剔除
- `F > 70` 且当前 F ≤ 70 → 剔除（对压制类）

---

## Phase 3 · 冲突评估

数据源：`经验冲突矩阵.yaml`

```
原子对冲突(a, b):
    查矩阵(a, b) → 严重度, 同章允许

读者状态冲突(a):
    查矩阵.读者状态冲突 where 经验ID==a
    逐条评估条件 → 累加严重度

Conflict(a) = max(原子对) + avg(读者状态冲突) × 0.5
```

---

## Phase 4 · 读者适配系数

查表法（`经验权重函数.md` 第三节）：

```
ReaderFit = 1.0
for each 命中规则 in 读者状态查表:
    ReaderFit ×= 规则.系数
clamp(ReaderFit, 0.2, 1.5)
```

---

## Phase 5 · Top3 贪心规则

1. 按 Score 降序遍历
2. 跳过 Conflict ≥ 70 的单条
3. 若与已选原子存在 `同章允许: false` → 跳过
4. 满 3 条停止
5. 若不足 3 条 → 从降级策略池补充（不违反禁止规则）

---

## Phase 6 · 四相映射

| 经验类别 | 四相 |
|----------|------|
| 压力 | 压制相 |
| 悬念、反转（未揭） | 触发相 |
| 高潮、爽点、情绪 | 爆发相 |
| 升级反思、悬念揭晓 | 收束相 |

同一经验只进入一个相；Top3 按类别自动分配。

---

## Phase 7 · 读者曲线预测

```
for each a in Top3:
    Δ += a.读者心理效果
应用衰减（见 04_读者模型/读者状态模型公式.md §3.2）
输出为变化量（±）
```

---

## 排序稳定性

同分情况下：

1. 优先「当前模式链下一步」对应的原子
2. 其次按经验ID字典序（保证可复现）
