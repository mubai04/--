# 回流 Xuke 快照接口

> Chuangjie 完章/完仿真后，向 Xuke 上报

---

## 上报时机

| 时机 | 说明 |
|------|------|
| 预演完成 | 仿真通过后上报，驱动下章决策 |
| 正文完章 | 复盘模式，对比实际与仿真 |
| 策略失效 | 偏差分 > 30，附带失败标记 |

## 上报格式

对齐 Xuke `10_决策系统/Snapshot快照层/snapshot.schema.yaml`：

```yaml
快照更新:
  章节ID: ""
  执行策略:
    模式链: ""
    使用经验: []
  结果:
    读者状态变化: { ... }
    成功得分: 0-100
  学习信号:
    加权经验: []
    降权经验: []
    新模式候选: []
```

## 成功得分计算（Chuangjie 侧）

```
成功得分 = 100 − 偏差分 − 沉浸中断分 × 0.5
```

Clamp 到 [0, 100]。

## 存储

本地副本写入 `07_Snapshot快照/记录/{章节ID}.yaml`，再推送 Xuke。
