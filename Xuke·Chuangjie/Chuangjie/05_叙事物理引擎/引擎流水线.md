```mermaid

  强度: 0-100
  参与角色: []
  标签: []
  时间序: 0
```

---

## Step 1 · 因果引擎

职责：建立事件触发链，保证「因→果」可追溯。

```
对每个事件 e:
  若 e.类型 依赖 前序事件 p → 建立 因果边 (p → e)
  因果强度 = min(p.强度, e.强度) × 因果系数（默认 0.8）
```

输出：`因果链展开[]`（写入仿真运行态）

---

## Step 2 · 冲突引擎

职责：计算冲突强度 C_t。

```
C_t = clamp(C_{t-1} + 基础冲突 + (P/100)×α_c + 矛盾密度×β_c + 目标冲突, 0, 100)
```

- 基础冲突：来自事件.强度
- 矛盾密度：未解决冲突数 / 总冲突数
- 目标冲突：角色目标对立程度 0–100

---

## Step 3 · 节奏引擎

职责：计算张力/节奏 T_pace，控制推进速度。

```
T_pace = σ(k × (C/100 − 0.5)) × 100 − 节奏瓶颈 + (X/100)×10
```

缓冲段：`节奏瓶颈 = 0`；连续高压无缓冲：`节奏瓶颈 = 15`

---

## Step 4 · 情绪引擎

职责：更新情绪波动 E 与爽点强度 SD。

```
E_t = clamp(E_{t-1} + ΔE_事件 + κ×(C/100 − P/100), −50, 50)
SD_t = clamp(SD_{t-1} + P×η1 − F_reader×η2, 0, 100)
```

---

## Step 5 · 状态更新

聚合四引擎输出 + 应用 `叙事物理公式.md` 全文：

- 更新世界运行时：P, SD, X, C, E, R
- 映射读者五维：Q, H, T, S, F
- 计算章节风险 R 与 Score
