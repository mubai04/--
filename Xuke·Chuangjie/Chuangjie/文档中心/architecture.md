# Chuangjie · 叙事运行时引擎（NRT Engine）

> **Chuangjie = 小说运行时系统（Narrative Runtime Engine）**  
> 不是写小说工具，而是：**把策略运行成剧情世界**

---

## 一、总架构（v2.0）

```text
Chuangjie（叙事执行引擎）
│
├─ 01_策略接收层        ← Xuke 决策输入
├─ 02_章节编译器        ← 策略 → 剧情结构（Narrative Compiler）
├─ 03_世界运行引擎      ← World Runtime
├─ 04_角色行为系统      ← Character Agent System
├─ 05_叙事物理引擎      ← Narrative Physics（可计算核心）
├─ 06_读者模拟器        ← Reader Simulation（执行层模拟）
├─ 07_冲突推进器        ← Conflict Driver（节奏调节）
├─ 08_章节输出层        ← 结构稿输出（非正文）
├─ 09_快照系统          ← Snapshot
├─ 10_反馈回流层        ← 回流 Xuke
└─ 11_接口层            ← 对外协议（Xuke 对接）
```

---

## 二、核心数据流

```text
Xuke 策略输入
    ↓
01 策略接收层
    ↓
02 章节编译器（策略 → 剧情结构）
    ↓
03 世界运行引擎（初始化/更新世界状态）
    ↓
04 角色行为系统（角色行动）
    ↓
05 叙事物理引擎（冲突/节奏/因果传播）
    ↓
06 读者模拟器（阅读体验预测）
    ↓
07 冲突推进器（节奏调节）
    ↓
08 章节输出层（结构稿）
    ↓
09 快照系统
    ↓
10 反馈回流层 → Xuke
```

---

## 三、系统本质

```text
Xuke      = 决策系统（应该怎么写）
Chuangjie = 运行系统（如果这样写，会发生什么）
```

```text
Chuangjie = 把叙事策略「运行成世界」的小说物理引擎
```

## 四、与 Xuke 指标对齐

| 层级 | 指标体系 |
|------|----------|
| Xuke 读者模型 | 期待值 Q、好奇心 H、紧张感 T、爽感储备 S、疲劳值 F |
| Chuangjie 世界物理 | 压制值 P、爽点强度 SD、悬念值 X、冲突强度 C、情绪波动 E、章节风险 R |

读者模拟器输出 **对齐 Xuke 五维**；叙事物理引擎维护 **世界运行时六维**。

公式详见 `05_叙事物理引擎/叙事物理公式.md`。
