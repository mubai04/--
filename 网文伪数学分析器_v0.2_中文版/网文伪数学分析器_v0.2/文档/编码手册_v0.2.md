# 编码手册 v0.2

## 证据对象

每条证据必须包含：

- `file`：必须属于当前任务 `documents`；
- `quote`：至少 8 个非空字符；
- `start`、`end`：对应章节文本中的精确字符位置；
- `reuse_justification`：同一证据支撑三个以上主判断时必填。

脚本会验证：

```text
对应章节正文[start:end] == quote
```

同一证据默认最多支撑两个主判断；写明复用理由后最多四个。

## 评分对象

有评分时：

```json
{
  "status": "OK",
  "min": 2.0,
  "max": 3.0,
  "evidence": [],
  "judgment": "判断说明"
}
```

数据不足时：

```json
{
  "status": "INSUFFICIENT_DATA",
  "evidence": [],
  "judgment": "缺少前章或样本不足"
}
```

不得把数据不足写成 0 分。

## 事实指标

比例类 `Q / ONE_MINUS_Q` 必须提供：

- `numerator >= 0`；
- `denominator > 0`；
- `numerator <= denominator`；
- `sample_size >= 0`；
- 原文证据和判断说明。

`N(x;a,b)` 类型必须提供非负 `value`。若注册表没有 C2 阈值，则只保留描述值，不进入聚合。

## 硬闸门

- `PASS`：材料充分且证据支持；
- `FAIL`：存在明确硬错误；
- `UNKNOWN`：材料不足或无法核验。

短文本不能直接 PASS：

- G_C、G_A、G_P：至少 300 字符；
- G_L：至少 500 字符；单章至少 800 字符。

## 最终状态机

- `BLOCKED`：硬闸门失败，或理解/因果上限不超过 1；
- `NEEDS_REVIEW`：硬闸门 UNKNOWN 或六维数据不足；
- `REWRITE`：章级能力上限低于 2.5，或任一风险达到 3；
- `TARGETED_REPAIR`：存在中风险、能力短板或反刷分失败；
- `ROBUST_PASS`：硬闸门全过，六维下限达标，九维风险均低于 2。
