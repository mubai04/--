# 方法验收与修订基线 v0.2

## 当前结论

方法可以作为个人内部分析项目运行，但统计有效性仍为 C0。v0.2 解决的是工程可信性，不等于完成同题材校准。

## 已落实修订

1. 正文目录和分析范围显式绑定；
2. 运行时执行 JSON Schema；
3. 四个硬闸门采用 `PASS / FAIL / UNKNOWN`；
4. 证据绑定真实任务文件、精确位置与逐字引文；
5. 非法数值、负分子、分子大于分母和非法区间直接失败；
6. `P_AI*` 改为 `AI_STYLE_INDEX`，禁止概率解释；
7. 不输出 `P_pureAI*`；
8. 支持 `NA / LOW_SAMPLE / NOT_APPLICABLE / INSUFFICIENT_DATA`；
9. `N(x;a,b)` 无 C2 阈值时不进入正式聚合；
10. 平滑化公式使用单调性正确传播区间；
11. 稳健通过条件包含 `E_ch`；
12. 期待与伏笔只计算已到期对象；
13. 相同病灶通过 `finding_id` 归并；
14. 增加六项反刷分检查；
15. 增加 Stability 权重扰动；
16. 增加最终状态机、版本比较和批处理失败传播；
17. 路径输入经过 resolve 和项目根目录约束。

## 方法分层

| 层级 | 含义 | 处理 |
|---|---|---|
| F0 | 字数、次数、位置、冲突、布尔事实 | 可直接核验 |
| F1 | 比率、密度、覆盖率 | 保留分子分母 |
| F2 | 0—4 代理风险 | 只作筛查与排序 |
| F3 | 人工/Codex 启发式编码 | 必须保留证据与判断 |
| F4 | 指数化展示 | 只作内部趋势 |
| X | 作者身份、商业成功率等越权结论 | 禁止 |

## 核心公式

```text
Q(p) = 4p，且 p 必须位于 [0,1]
N(x;a,b) = clip(4 × (x-a)/(b-a), 0, 4)，b>a
WA = 仅对适用且样本充分的项目重归一加权
```

本项目不再对非法比例静默裁剪。

## 三态闸门

```text
任一 FAIL             → BLOCKED
无 FAIL，存在 UNKNOWN → NEEDS_REVIEW
全部 PASS             → PASSED
```

短材料存在强制适用性阈值，不能凭一两句正文把硬闸门判为 PASS。

## 最终状态

- `BLOCKED`：硬错误；
- `NEEDS_REVIEW`：关键数据不足；
- `REWRITE`：结构能力过低或风险达到高位；
- `TARGETED_REPAIR`：局部风险、能力短板或反刷分失败；
- `ROBUST_PASS`：硬闸门全过、六维下限达标、九维风险均低。

## 校准等级

- C0：定义、初始权重和边界；
- C1：稳定区分同章版本；
- C2：同题材、平台、篇幅内有效；
- C3：与盲评或真实行为稳定相关。

当前固定为 C0，不得外推。
